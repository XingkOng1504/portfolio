Présentation personnelle

Mon projet professionnel

Mon orientation vers le développement Back-End n'est pas le fruit du hasard : elle découle d'une appétence concrète pour l'automatisation des flux d'information, notamment dans les secteurs de l'e-commerce et de la logistique.

Durant mon stage, j'ai pu mesurer l'impact direct d'un code robuste sur des processus critiques : la génération automatique de bordereaux d'expédition, la synchronisation des catalogues produits entre marketplaces. Ce type de problème, à l'intersection du code et du métier, est exactement ce qui me motive.

Mon objectif est d'intégrer le diplôme d'ingénieur Informatique et Systèmes d'Information du Cnam en alternance, pour ancrer mes compétences techniques dans une réalité d'entreprise dès la troisième année.

Pourquoi l'informatique ?

Ce qui m'a attiré dans l'informatique, c'est la capacité à transformer un problème complexe en une solution reproductible. Lorsqu'une tâche répétitive peut être automatisée, on libère du temps pour des décisions à plus haute valeur ajoutée.

Cette logique s'est confirmée dès mes premières SAÉ : j'y ai trouvé plus d'intérêt à comprendre pourquoi une architecture fonctionne qu'à simplement la faire tourner. C'est cette posture analytique que je cherche à approfondir vers un rôle d'ingénieur logiciel orienté systèmes.

SAÉ S4 · Projet universitaire n°2

Mnémosyne

Plateforme de suivi des cohortes étudiantes — IUT Villetaneuse

Flask Python SQLite3 ScoDoc API Sankey Docker

Contexte & objectif

Depuis la réforme BUT 2021, l'IUT Villetaneuse (6 départements, 3 années) peine à visualiser les flux d'étudiants entre promotions. Mnémosyne répond à ce besoin via des diagrammes de Sankey interactifs connectés à l'API ScoDoc, permettant de suivre passages, redoublements et abandons.

Ce que j'ai apporté

  • Architecture MVC Flask avec séparation stricte Controllers / Services / DAO
  • Système de règles métier configurables pour qualifier les parcours étudiants
  • Export PDF des rapports de cohorte
  • Conteneurisation complète via Docker & docker-compose
  • Données anonymisées JSON (conformité RGPD)
Preuve de compétence

La mise en place du diagramme de Sankey illustre ma capacité à traiter et visualiser des données complexes : il a fallu transformer des décisions de jury brutes (ADM, PASD, RED, NAR, ADJ) en flux agrégés cohérents, puis les lier à une interface permettant de "percer" chaque flux pour consulter les détails individuels.

Technologies Python · API REST · UPS · Packlink · Octopia · Cdiscount · Back Market

Ce que j'ai découvert du monde professionnel

L'entreprise m'a confié le développement de scripts Python d'automatisation pour des processus critiques : génération de bordereaux d'expédition (UPS, Packlink), synchronisation de catalogues produits avec les marketplaces partenaires (Octopia, Cdiscount, Back Market).

Cette expérience m'a appris que le monde professionnel impose des contraintes que l'université simule difficilement :

  • La robustesse prime sur l'élégance : un script qui crashe en production sur un bordereau d'expédition coûte de l'argent réel.
  • Les infrastructures héritées sont contraignantes : travailler sur un environnement PC/Windows avec des bases Alwaysdata impose des adaptations que les tutos en ligne n'anticipent pas.
  • La documentation des API tierces est rarement complète : décortiquer les specs d'Octopia ou Packlink m'a demandé plus de temps que le développement lui-même.

Compétences BUT mobilisées

Réaliser le développement d'applications

Les scripts Python conçus à la DSI vont au-delà du simple appel API : gestion des erreurs réseau, retry automatique, logging structuré et production d'un rapport d'exécution lisible par les équipes métier. J'ai dû produire un code testable et maintenable par d'autres, pas seulement par moi.

Optimiser des applications informatiques

Face au volume de commandes à traiter (appels API en boucle), j'ai identifié des goulots d'étranglement et mis en place un mécanisme de traitement par lots (batch processing), réduisant significativement le temps total d'exécution et les risques de rate-limiting côté API.

Regard critique sur le stage

Ce stage a confirmé mon orientation tout en en révélant les limites : j'ai opéré seul sur des scripts sans revue de code systématique ni suite de tests. Cette autonomie m'a appris à prendre des décisions, mais elle a aussi mis en lumière l'importance des pratiques collaboratives (revue de code, CI/CD) que je n'ai pas encore pu expérimenter en entreprise. C'est précisément ce que l'alternance visée au Cnam devra m'apporter.
